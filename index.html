    QuickMeet - Real-Time Video Meeting App

    Description

    QuickMeet is a modern video conferencing application designed to provide a seamless and secure online meeting experience. Users can create or join virtual meeting rooms instantly by entering a unique room code. It supports peer-to-peer video and audio calls, in-room chat, and real-time user presence. The project was built to demonstrate expertise in WebRTC, real-time communication, and scalable backend design.

    Key Features

    • Create or join meeting rooms using a unique meeting code
    • Real-time video and audio communication powered by WebRTC
    • Live signaling, room management, and user presence with Socket.IO
    • In-room chat messaging for text communication
    • Lightweight and responsive UI for smooth user experience
    • Automatic handling of multiple participants in a single room
    • Secure communication with HTTPS and authentication flow
    • Cross-browser compatibility and mobile responsiveness

    Technologies

    Frontend: React.js, HTML5, CSS3, JavaScript
    Backend: Node.js, Express.js
    Database: MongoDB Atlas
    Real-Time Communication: WebRTC, Socket.IO
    Others: JWT Authentication, REST APIs

    Challenges & Solutions

    Challenge 1: Establishing reliable peer-to-peer video and audio streams between multiple participants.
    Solution: Implemented WebRTC with ICE candidates and STUN/TURN servers to handle network traversal and NAT issues.

    Challenge 2: Managing real-time signaling and synchronization when users join/leave rooms.
    Solution: Used Socket.IO for event-based signaling and dynamic room management, ensuring smooth participant updates.

    Challenge 3: Ensuring secure and smooth meeting flow across devices.
    Solution: Added JWT-based authentication, HTTPS setup, and responsive design for cross-device compatibility.

    Airbnb - Property Booking Platform

    Description

    This Airbnb Clone allows users to browse, search, and filter property listings easily. Users can log in to list their own properties, edit or delete them, and manage reviews. Maps integration helps users select and view property locations with markers. Full authentication, authorization, and error handling ensure a secure and smooth experience.

    Key Features

    • User authentication with JWT – login/register required for listing or reviews
    • Search and filter properties by location, price, type, and amenities
    • Users can list their own properties and edit or delete only their listings
    • Users can add reviews to properties and delete their own reviews
    • Map integration using Mapbox – property location displayed with markers
    • Non-logged-in users can only view listings, cannot create listings or reviews
    • Proper error handling with user-friendly error messages
    • Responsive UI for mobile and desktop

    Technologies

    Frontend: EJS, HTML5, CSS3, Tailwind CSS, Lightswind
    Backend: Node.js, Express.js
    Database: MongoDB, Mongoose, Connect-Mongo
    Authentication: Passport.js, JWT, Passport-Local-Mongoose
    File Upload & Cloud: Multer, Cloudinary, Multer-Storage-Cloudinary
    Other: Mapbox SDK, Method-Override, Connect-Flash, Joi Validation

    Challenges & Solutions

    Challenge 1: Allowing only owners to edit/delete their listings and reviews.
    Solution: Implemented JWT-based authentication with proper authorization checks.

    Challenge 2: Handling location selection and display on a map.
    Solution: Integrated Mapbox SDK with property markers for interactive map experience.

    Challenge 3: Ensuring smooth error handling for multiple actions (listing, reviews, login).
    Solution: Created centralized error handling with descriptive popup messages using Connect-Flash.

    Doctor–Patient Management System

    Description

    Doctor–Patient Management System is a role-based healthcare portal designed to simplify the management of doctors and patients. The application provides an Admin Panel to add, update, and delete doctor and patient records, while regular Users can log in to view available doctors and patient details. Built with Java Servlets, JDBC, and SQL, this project demonstrates strong backend integration, authentication, and database management skills.

    Key Features

    • 🔹 Role-based authentication for Admin and User access
    • 🔹 Admin Dashboard to manage doctor and patient records (CRUD)
    • 🔹 User Portal to view available doctors and patient list
    • 🔹 Secure Login System with session handling
    • 🔹 SQL Database Integration for storing and managing healthcare data
    • 🔹 Responsive UI built with HTML5, CSS3, and JavaScript

    Technologies

    Frontend: HTML5, CSS3, JavaScript
    Backend: Java Servlets, JDBC
    Database: MySQL / SQL
    Others: JSP, Session Management, MVC Architecture

    Challenges & Solutions

    Challenge 1: Implementing role-based access control for admin and users.
    Solution: Designed a login system with session-based authentication and access filters to separate admin and user dashboards.

    Challenge 2: Managing relational data between doctors and patients.
    Solution: Used SQL relational tables with foreign keys to link doctors and patients efficiently.

    Challenge 3: Ensuring data integrity during CRUD operations.
    Solution: Applied prepared statements in JDBC to prevent SQL injection and maintain data security.
